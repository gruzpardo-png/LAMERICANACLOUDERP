Regla para no perder datos al actualizar

Las actualizaciones de software deben reemplazar código, plantillas y archivos estáticos, pero no deben cambiar la variable DATABASE_URL ni eliminar la base PostgreSQL de Render.

  • No borres la base PostgreSQL en Render.
  • No cambies el DATABASE_URL por una base nueva si quieres conservar la información.
  • No agregues código con drop_all(), eliminación masiva o reinicio de tablas.
  • db.create_all() solo crea tablas faltantes; no borra datos existentes.
